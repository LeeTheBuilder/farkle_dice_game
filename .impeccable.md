## Design Context

### Users
This is a local hot-seat dice game for two people sharing one screen, with an optional browser-run AI opponent. Players are using it casually at home and need fast, readable table-state feedback without setup friction.

### Brand Personality
Medieval tavern, tactile, spirited. The interface should feel like dice on felt and wood, not like a generic game menu or casino app.

### Aesthetic Direction
Use the existing tavern-table direction: dark wood, green felt, brass accents, parchment surfaces, and original CSS dice. Motion should make dice rolls and turn changes clearer, while staying lightweight enough for mobile and file-based local play.

### Design Principles
- Let the dice be the hero moment; gameplay state should be visible before players make decisions.
- Keep motion short, readable, and useful, with reduced-motion support.
- Preserve the no-dependency static app model.
- Favor tactile feedback over decorative effects.
